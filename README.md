# TRABALHO-AVALIATIVO-TDD
## Objetivos
- Definir o teste
- Executar o teste (falha esperada)
- Implementar o código
- Executar o teste novamente
- Refatorar o código (correções, melhorias e tratamento de erros e de limitações)

## METODOLOGIA
A metodologia adotada neste trabalho baseou-se na prática de Test-Driven Development (TDD) aplicada ao desenvolvimento de três operações básicas: soma, subtração e divisão. Para a implementação do código, foi utilizada a linguagem de programação Python, juntamente com o framework de testes Pytest. Além disso, foi empregado o GitHub como ferramenta de controle de versão e organização do projeto.
Inicialmente, foram definidos os requisitos de cada operação, identificando os comportamentos esperados e possíveis casos de erro. Em seguida, foram elaborados testes automatizados para cada operação, seguindo o princípio do TDD, no qual os testes são escritos antes da implementação do código-fonte. Esses testes foram versionados e armazenados em um repositório no GitHub.
Após a criação dos testes, o código das operações foi desenvolvido de forma incremental, com o objetivo de satisfazer todos os testes previamente definidos. A cada etapa do desenvolvimento, as alterações foram registradas por meio de commits no GitHub, permitindo o acompanhamento da evolução do projeto e o histórico das modificações realizadas. Os testes foram executados continuamente utilizando o Pytest, possibilitando a validação constante das funcionalidades implementadas.
Por fim, foram analisados os resultados obtidos, destacando a eficácia do TDD no desenvolvimento das operações propostas, bem como a importância do uso do GitHub para o controle de versão, colaboração e manutenção do software.
